Typical Development Tasks

  1. Command Line:


    1. Compile the Open JBI Components source:

      % cd $JV_SRCROOT
% smvn clean
% smvn

      Note: You are only required to run the "smvn clean" command the first time you do the build. After the first time you run the build you can just run "smvn" or "smvn clean"

      Note: If you use a web proxy server to connect to the Internet you need to create a settings.xml file that specifies the proxy settings. The location of this file is ${user.home}/.m2/settings.xml. Details on the content of this file are located on the build instructions FAQ page Here.


    2. Run the junit tests:

      % cd $JV_SRCROOT
% smvntest

      A junit report will be created under $JV_SRCROOT/bld/junit/html


    3. Working in a sub-project:

      You can cd into any project directory (for example bpelse) and just compile and test that directory. For example:

      % cd $JV_SRCROOT/ojc-core/bpelse
% smvn
% smvntest
